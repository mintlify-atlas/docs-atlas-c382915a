## Overview

The `config` command manages acpx configuration files. Configuration is loaded from global and project files, with CLI flags taking precedence.

## Syntax

```bash
acpx config [subcommand]
acpx config show
acpx config init
```

## Subcommands

### show

Display the resolved configuration from global and project files.

```bash
acpx config show
acpx config  # alias for show
```

**Output:**
- Resolved configuration values
- File paths (global and project)
- Which files were loaded

### init

Create a default global configuration file if it doesn't exist.

```bash
acpx config init
```

**Behavior:**
- Creates `~/.acpx/config.json` if missing
- Does not overwrite existing file
- Populates with default configuration template

## Configuration Files

Configuration is read in this order (later wins):

1. **Global**: `~/.acpx/config.json`
2. **Project**: `<cwd>/.acpxrc.json`

CLI flags always override config values.

## Configuration Schema

```json
{
  "defaultAgent": "codex",
  "defaultPermissions": "approve-all",
  "nonInteractivePermissions": "deny",
  "authPolicy": "skip",
  "ttl": 300,
  "timeout": null,
  "format": "text",
  "agents": {
    "my-custom": { "command": "./bin/my-acp-server" }
  },
  "auth": {
    "my_auth_method_id": "credential-value"
  }
}
```

### Configuration Keys

<ParamField path="defaultAgent" type="string" default="codex">
  Default agent to use when no agent is specified.
</ParamField>

<ParamField path="defaultPermissions" type="string" default="approve-reads">
  Default permission mode: `approve-all`, `approve-reads`, or `deny-all`.
</ParamField>

<ParamField path="nonInteractivePermissions" type="string" default="deny">
  Non-TTY permission policy: `deny` or `fail`.
</ParamField>

<ParamField path="authPolicy" type="string" default="skip">
  Authentication policy for ACP adapters.
</ParamField>

<ParamField path="ttl" type="number" default="300">
  Queue owner idle TTL in seconds. `0` disables TTL.
</ParamField>

<ParamField path="timeout" type="number" default="null">
  Default timeout for agent responses in seconds. `null` means no timeout.
</ParamField>

<ParamField path="format" type="string" default="text">
  Default output format: `text`, `json`, or `quiet`.
</ParamField>

<ParamField path="agents" type="object">
  Custom agent command mappings. Keys are agent names, values contain `command` strings.
</ParamField>

<ParamField path="auth" type="object">
  Authentication credentials for ACP adapters. Keys are auth method IDs.
</ParamField>

<ParamField path="disableExec" type="boolean" default="false">
  Disable the `exec` subcommand entirely.
</ParamField>

<ParamField path="queueMaxDepth" type="number">
  Maximum queue depth for prompt queueing.
</ParamField>

<ParamField path="mcpServers" type="object">
  MCP server configuration (advanced).
</ParamField>

## Examples

### Show Configuration

```bash
$ acpx config show
{
  "defaultAgent": "codex",
  "defaultPermissions": "approve-reads",
  "nonInteractivePermissions": "deny",
  "authPolicy": "skip",
  "ttl": 300,
  "timeout": null,
  "format": "text",
  "agents": {},
  "auth": {},
  "paths": {
    "global": "/home/user/.acpx/config.json",
    "project": "/home/user/project/.acpxrc.json"
  },
  "loaded": {
    "global": true,
    "project": false
  }
}
```

### Initialize Configuration

```bash
$ acpx config init
Created /home/user/.acpx/config.json
```

If file already exists:

```bash
$ acpx config init
Config already exists: /home/user/.acpx/config.json
```

### JSON Output

```bash
# Show as JSON
acpx --format json config show

# Init as JSON
acpx --format json config init
# {"path":"/home/user/.acpx/config.json","created":true}
```

### Quiet Output

```bash
$ acpx --format quiet config init
/home/user/.acpx/config.json
```

## Configuration Examples

### Custom Default Agent

```json
{
  "defaultAgent": "claude"
}
```

Now `acpx "prompt"` uses `claude` instead of `codex`.

### Auto-Approve All Permissions

```json
{
  "defaultPermissions": "approve-all"
}
```

All commands default to `--approve-all` behavior.

### Custom Agent Command

```json
{
  "agents": {
    "openclaw": {
      "command": "env OPENCLAW_HIDE_BANNER=1 OPENCLAW_SUPPRESS_NOTES=1 node scripts/run-node.mjs acp --url ws://127.0.0.1:18789 --token-file ~/.openclaw/gateway.token --session agent:main:main"
    }
  }
}
```

Overrides the built-in `openclaw` command with a custom local checkout.

### Custom Agent

```json
{
  "agents": {
    "my-agent": {
      "command": "./bin/my-custom-acp-server"
    }
  }
}
```

Now `acpx my-agent "prompt"` works as a built-in agent.

### Authentication Credentials

```json
{
  "auth": {
    "api_key": "sk-...",
    "oauth_token": "token-..."
  }
}
```

Credentials passed to ACP adapters during authentication.

### Short TTL for Development

```json
{
  "ttl": 30
}
```

Queue owners shut down after 30 seconds of idle time.

### Disable TTL (Keep Alive)

```json
{
  "ttl": 0
}
```

Queue owners never shut down due to idle timeout.

### Default Timeout

```json
{
  "timeout": 120
}
```

All commands default to 120-second timeout.

### JSON Output by Default

```json
{
  "format": "json"
}
```

All commands output JSON by default (unless overridden with `--format`).

### Disable exec

```json
{
  "disableExec": true
}
```

The `exec` subcommand will be disabled:

```bash
$ acpx exec "query"
Error: exec subcommand is disabled by configuration (disableExec: true)
```

## Project Configuration

Create `.acpxrc.json` in your project root:

```json
{
  "defaultPermissions": "approve-all",
  "agents": {
    "dev": {
      "command": "./scripts/dev-acp-server.sh"
    }
  }
}
```

Project config merges with global config, with project values taking precedence.

## Configuration Precedence

From lowest to highest priority:

1. Built-in defaults
2. Global config (`~/.acpx/config.json`)
3. Project config (`<cwd>/.acpxrc.json`)
4. CLI flags

### Example Precedence

Global config:
```json
{
  "defaultAgent": "codex",
  "ttl": 300
}
```

Project config:
```json
{
  "defaultAgent": "claude",
  "defaultPermissions": "approve-all"
}
```

Command:
```bash
acpx --ttl 60 "prompt"
```

Resolved:
- `defaultAgent`: `"claude"` (project overrides global)
- `ttl`: `60` (CLI flag overrides all)
- `defaultPermissions`: `"approve-all"` (from project)

## set Command

The `set` command sets session config options (not acpx configuration).

### Syntax

```bash
acpx [global_options] <agent> set <key> <value> [-s <name>]
acpx [global_options] set <key> <value> [-s <name>]  # defaults to codex
```

### Behavior

1. Finds the session matching scope `(agentCommand, cwd, name?)`
2. If no session exists, exits with code `4`
3. Routes through queue-owner IPC if active, otherwise reconnects directly
4. Calls ACP `session/set_config_option` with the specified key/value
5. Config option keys and values are adapter-defined

### Examples

```bash
# Create session
acpx codex sessions new

# Set config option
acpx codex set model gpt-4
acpx codex set temperature 0.7
acpx codex set max_tokens 2000
```

### Codex Compatibility Alias

`acpx` automatically translates `thought_level` to `reasoning_effort` for Codex:

```bash
# These are equivalent for codex
acpx codex set thought_level high
acpx codex set reasoning_effort high
```

### Named Session

```bash
acpx codex set model gpt-4o -s backend
```

### Output Formats

**Text:**
```bash
$ acpx codex set model gpt-4

```

**JSON:**
```bash
$ acpx --format json codex set model gpt-4
{
  "action": "config_set",
  "configId": "model",
  "value": "gpt-4",
  "resumed": false,
  "configOptions": [...],
  "acpxRecordId": "01JQWE...",
  "acpxSessionId": "01JQWF...",

```

**Quiet:**
```bash

```
