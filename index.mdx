## Why acpx?

Your agents love acpx! They hate having to scrape characters from a PTY session. `acpx` provides a structured protocol for AI agents and orchestrators to communicate with coding agents over ACP instead of terminal scraping.

<CardGroup cols={2}>
  <Card title="Persistent Sessions" icon="clock-rotate-left">
    Multi-turn conversations that survive across invocations, scoped per repository
  </Card>
  <Card title="Named Sessions" icon="layer-group">
    Run parallel workstreams in the same repo with `-s backend` and `-s frontend`
  </Card>
  <Card title="Prompt Queueing" icon="list">
    Submit prompts while one is already running — they execute in order automatically
  </Card>
  <Card title="Structured Output" icon="code">
    Typed ACP messages (thinking, tool calls, diffs) instead of ANSI scraping
  </Card>
</CardGroup>

## Quick Example

```bash
# Create a session and delegate work to a coding agent
acpx codex sessions new
acpx codex "find the flaky test and fix it"

# Work in parallel named sessions
acpx codex -s api "implement token pagination"
acpx codex -s docs "rewrite API docs"

# One-shot execution without saved session
acpx codex exec "summarize this repo"
```

## Supported Agents

acpx works with multiple ACP-compatible coding agents out of the box:

- **Pi** — Pi Coding Agent via pi-acp adapter
- **OpenClaw** — Native ACP bridge (`openclaw acp`)
- **Codex** — Codex CLI via codex-acp adapter
- **Claude** — Claude Code via claude-agent-acp adapter
- **And more** — Gemini, Cursor, Copilot, and custom agents
